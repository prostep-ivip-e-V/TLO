Ontology Specification Draft

Abstract

Increasing demand for interdisciplinary interaction in product and system lifecycles raises the need for cross project collaboration and common results. Driven by the industrial need, various prostep ivip (psi) project groups are dealing with future collaboration scenarios. Therefore, focused collaboration processes, information models and technologies are defined individually. The vision of the psi Ontology initiative is to enable and support collaboration scenarios based on a harmonized reference model combining a unified information model (semantic + structure) and validated technology. Based on an ontological approach, the aim is to create an information model across project groups that represents commonalities, connections, but also gaps between the individual project groups and makes the information gained reusable. This documentation gives details of the developed psi top-level ontology.

Introduction back to ToC

The psi ontology initiative was launched in autumn 2021 with the goal of an integrated knowledge management across the different project groups. The first step in creating the psi Ontology was gathering and analysing the available input of the individual working groups. In particular, the focus lay on use cases, (reference) processes and data models, as well as standards used and the already known collaborations with other project groups. Based on this input an abstraction was conducted over the information of the project groups and an ontology-based prostep ivip reference model was created. Freely available ontologies were included what ensures usability and extensibility of the resulting ontology.
